01Premise

Hybrid retrieval, zero runtime infra.

Deadwater is a research publication index with a hybrid retrieval engine. BM25 lexical scoring combines with dense embedding similarity for best-of-both-worlds ranking. Phrase search, domain filtering, and pagination are built in; a query cache makes repeats instant.

The Python package builds the index. A Node.js generator emits ~2,000 article pages plus archives, publications, monthly newsletters, sitemaps, and Atom/RSS feeds. The runtime is fully static — a service worker intercepts API calls so search works offline.

03Quickstart

Build index, generate site, serve static.

Index in Python, generate in Node, serve static. The service worker is the only "runtime" in production.

# Prereqs: Python 3.10+, Node.js 20+, pip
$ git clone https://github.com/Real-Fruit-Snacks/Deadwater.git
$ cd Deadwater
$ pip install -e ".[dev]"
$ node build.js
$ cd public && python3 -m http.server 8000

# Test search API
$ curl http://localhost:8000/api/v2/search?q=distributed+consensus

# Run test suite
$ pytest

05Architecture

Python engine, Node generator.

The Python core handles BM25 + embeddings. The Node generator emits all the static output. The service worker handles API calls offline by serving the same JSON.

build.js                      // Static site generator (Node)
pyproject.toml                // Python package configuration

src/deadwater/
core/
  engine.py                   // Hybrid search (BM25 + semantic)
  index.py                    // Inverted index with BM25 scoring
models/
  article.py                  // Article models with citation export
  author.py                   // Author profiles

public/                       // Generated static site
index.html                    // Main page with infinite scroll
sw.js                         // Service worker for offline + API
research/                     // Article pages
newsletter/                   // Monthly digest archive
api/docs/                     // OpenAPI 3.1 specification
citations/                    // BibTeX and RIS files
datasets/                     // CSV and JSON exports
06Closing

Static at rest, hybrid at search time.

No databases, no containers, no cloud functions in production. Push to main triggers GitHub Pages deploy; a nightly workflow regenerates content to keep it current.
