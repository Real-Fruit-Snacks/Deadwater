Core Features

Search, index, and serve research publications. Python engine builds the index, Node generates the site, GitHub Pages serves it.

🔍

Hybrid Search

BM25 lexical scoring combined with dense embedding similarity. Phrase search, domain filters, pagination.

engine.search("distributed consensus", domains=[...])
📄

Static Site Generator

2,000+ article pages, 200 archive pages, 36 newsletters, sitemaps, feeds. All SEO-optimized.

node build.js # generates public/
🔌

REST API

OpenAPI 3.1 spec. Paginated article listing, full-text search, single paper retrieval, bulk export.

GET /api/v2/search?q=neural+arch&type=all
⚖

GraphQL

Cursor-based pagination. Query articles by domain, author, citation count. Nested relation traversal.

{ articles(domain: "...", first: 10) { edges } }
📚

Citation Export

BibTeX, RIS, CSV, JSON, JSONL. Single-paper and bulk. Standards-compliant for Zotero, Mendeley, EndNote.

GET /api/v2/papers/42?format=bibtex
📡

Newsletter Archive

Three years of monthly research digests. Curated highlights, trending papers, domain summaries.

36 issues | /newsletter/2024-01/ ... /2026-12/

Technology

Python for search intelligence. Node for site generation. Service workers for offline-first delivery.

BM25 Scoring

Inverted index with term frequency, document length normalization, and IDF weighting. Phrase search with positional index support.

Semantic Embeddings

Dense vector representations (768-dim) for meaning-based retrieval. Optional GPU acceleration via CUDA, ROCm, or Metal.

Service Worker API

Intercepts /api/v2/* requests and serves pre-built JSON. Search works offline. IndexedDB-backed query cache for instant repeats.

OpenAPI 3.1

Full REST specification with schema definitions, examples, and error responses. Auto-generated documentation at /api/docs/.

Static Generation

Node.js build produces 2,000+ HTML pages with structured data, Open Graph meta, and XML sitemaps. Fully crawlable by search engines.

Citation Standards

BibTeX and RIS output validated against Zotero, Mendeley, and EndNote. CSV and JSON for data analysis. JSONL streaming for bulk.

Query Cache

LRU cache with configurable size (default 10,000 entries). Persisted to IndexedDB for cross-session access. Sub-millisecond repeat queries.

Domain Taxonomy

43 research domains from distributed systems to quantum computing. Hierarchical classification with cross-domain article tagging.

RSS + Atom

Standards-compliant feeds for research updates. Subscribe in any feed reader. Auto-generated during each build cycle.

Platform Capabilities

Python search engine runs on any platform. Generated site serves from any static host. APIs work offline via service worker.

Architecture

Three-stage pipeline. Python indexes, Node generates, GitHub Pages serves. Zero runtime infrastructure.

1

Search Engine

Python package with hybrid BM25 + semantic similarity. Inverted index with phrase support. Configurable embedding dimensions. LRU query cache.

2

Data Models

Article and Paper models with citation export (BibTeX, RIS). Author profiles with impact metrics. Domain taxonomy with 43 research categories.

3

Site Generator

node build.js produces 2,000+ article pages, archive pages, newsletters, feeds, sitemaps, and citation files. All static HTML + JSON.

4

Service Layer

Service worker intercepts API requests, serves pre-built JSON. Offline-first with IndexedDB cache. GraphQL resolved client-side against cached data.

Getting Started

Install the Python package, build the static site, serve it locally or deploy to GitHub Pages.

build.sh 
# Clone and install
$ git clone https://github.com/Real-Fruit-Snacks/Deadwater.git
$ cd Deadwater
$ pip install -e ".[dev]"

# Build static site
$ node build.js
Generated 2,000 article pages
Generated 200 archive pages
Generated 36 newsletter issues
Generated 6 sitemaps + index

# Serve locally
$ cd public && python3 -m http.server 8000
Serving on 0.0.0.0:8000
search.py 
# Python search API
>>> from deadwater import ResearchEngine
>>> engine = ResearchEngine("./data/index")
>>> engine.load()

>>> r = engine.search("distributed consensus")
>>> for hit in r.results[:3]:
...     print(f"{hit.score:.3f} | {hit.article.title}")
0.847 | Raft: Understandable Consensus
0.812 | Paxos Made Simple
0.798 | Byzantine Fault Tolerance

# REST API (served by service worker)
$ curl /api/v2/search?q=neural+architecture
{"results": [...], "total": 142, "page": 1}
